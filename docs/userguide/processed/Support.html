1.4 Documentation and Support
Where to find Help and Support for McIDAS-V
If you are having problems installing or using McIDAS-V, or want to know how to perform a particular task, you have several resources to help you out.
1.4.0  McIDAS-V Documentation
Extensive documentation is available for McIDAS-V
1.4.1  McIDAS-V Mailing Lists
McIDAS-V email lists, "idvusers" and "idvdevelopers" put you in contact with other IDV users and developers
1.4.2  Getting help from Unidata Support
For qualified users, the Unidata Program Center provides direct support via email.
1.4.0 McIDAS-V Documentation

The first source of support is this McIDAS-V User's Guide. It contains complete instructions for downloading, installing, running, and using the McIDAS-V reference application and all its features. The User's Guide can be accessed from the McIDAS-V Help->User's Guide menu or online at:


http://www.unidata.ucar.edu/software/idv/docs/userguide/index.html

Be sure to check the McIDAS-V Frequently Asked Questions (FAQ) and the Known Problems pages of the User's Guide to see if your question can be answered there.

Additional documentation is available on the McIDAS-V homepage:

http://www.unidata.ucar.edu/software/idv/index.html

including:

  • Unidata IDV Training Workshop - serves as a tutorial.
  • Javadoc for McIDAS-V source code - serves as developer documentation.
  • IDV Support e-mail archive - searchable archives of IDV questions answered by the Unidata User Support team.
There is also a gallery of examples displays created in McIDAS-V.
1.4.1 IDV Mailing Lists

"idvusers" is a list for all IDV users. The intended topics of discussion are any matters pertaining to use of the IDV reference application, downloading and running McIDAS-V, data sources and formats for use in McIDAS-V, and sharing of IDV based materials (bundles, educational materials, etc). This list is used by the Unidata Program Center to announce new releases of McIDAS-V.

"idvdevelopers" is a discussion email list for IDV developers who are using IDV source code to build customized or new IDV-based applications. The software developers of McIDAS-V are subscribers to this list and may be able to help with questions about use of McIDAS-V software framework.

You must be subscribed to the list before you can post to it. Subscribe to the email lists at : http://www.unidata.ucar.edu/support/mailinglist/mailing-list-form.html.

1.4.2 Getting help from Unidata Support

Unidata provides direct support via email for its software packages to institutions offering college-level courses for credit, and to non-profit institutions with education or research missions.

Note: there are conditions on accessing or obtaining support.

If you are unable to find the answers to your IDV questions/problems in the references above and you are a member of a qualified institution outlined above, you can contact Unidata User Support and request help.

There are 3 methods which you can use to contact the Unidata User Support:

  1. Fill out the support form available from the Help->Support Request Form... menu. Please include as much information as possible about the problem. (See Below).

  2. Fill out the support form available on the Unidata Support page:

    http://www.unidata.ucar.edu/support/email_support.php

    Please include as much information as possible about the problem. (See Below).

  3. If you cannot use the forms above, you can send a message directly to support-idv@unidata.ucar.edu Please use this email address only for IDV related questions.

Questions and answers to Unidata User Support are archived so there is a record of past solutions to problems. If you do not wish your email to be in the public domain please say so in each email.

Please look at the McIDAS-V FAQ and the list of Known Problems before reporting a problem. Please read "Reporting Errors to Unidata Support" below before sending in a report.

Reporting Errors to Unidata Support

If you experience unexpected behavior from McIDAS-V other than the known problems listed in the Known Problems section, please let us know.

Check for related error messages in the McIDAS-V Console, and include the appropriate error messages in your message to us.

Also include as much information as you can about how you were running McIDAS-V and what happened:

  • the version of McIDAS-V that you are using (use the Help->About... menu.)
  • how you download and run McIDAS-V (InstallAnywhere, Web Start, or built from source)
  • your operating system and version level
  • any messages on the console (to see the console, use the Help->Show Console menu.)
  • exact data source and parameter choice
  • types of display being made and in use
  • the sequence of McIDAS-V actions and choices that caused the problem
  • A bundle file that recreates the problem if the data sources are network accessible (optional, but desirable).
Send this information to Unidata User Support using one of the methods outlined above.

Support Response

We make every effort to reply to McIDAS-V questions in a timely manner. Because Unidata's primary mission is to serve geo-scientists and students at U.S. colleges and universities, problems reported by that community necessarily receive first attention.
