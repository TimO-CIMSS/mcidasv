User Preferences - Advanced

The User Preferences window is accessed through the Edit->Preferences menu. The Advanced tab allows the user to change options that require a restart in order for them to take effect. This tab can also be accessed by using the startup manager to configure settings before running McIDAS-V (runMcV-Config.bat or runMcV-Config). The below image represents the default settings for the Advanced tab.


Image 1: Advanced Preferences (Default)

Startup Options

Memory - When the first bubble is selected McIDAS-V will use up to the given amount of the computer's available memory. When the second bubble is checked, the user can enter a specific set value of memory to use. The new amount of memory will be saved and used in subsequent sessions. For 32 bit operating systems, it is recommended to set this to no more than 1250 MB. The maximum value for 32 bit operating systems is 1536 MB. However, 64 bit operating systems can use all of the memory available to the operating system.

Specify Default Bundle - When checked, select a bundle from your list of favorites (found in the Bundles menu) to load upon startup.

Load Default Layout - When checked, McIDAS-V will load the default layout upon startup. Uncheck to use the default layout of McIDAS-V (one window with one panel).

Enable 3D controls - When checked, McIDAS-V uses the default 3D behavior. Uncheck to override the default 3D behavior of McIDAS-V and use a 2D View for Map Views.

Enable JOGL - Java OpenGL (JOGL) is a wrapper library that allows OpenGL to be used in Java, and it is permantely enabled in McIDAS-V.

Enable Direct3D - When checked, Direct3D will be used. Direct 3D is used to render 3D graphics. Note: this only works in Windows.

Enable Geometry by Reference - When checked, Java Geometry by Reference will be used. Try this option unchecked if random/extraneous map lines are drawn - a problem that may occur more frequently on systems with ATI graphics cards.

Enable Access to Image Data by Reference - When checked, Java Access to Image Data by Reference will be used. We recommend that this option be checked, as it often removes the jagged edges at the limb and/or gaps at the dateline that sometimes appear in displays of image data.

Enable Non-Power of Two (NPOT) Textures - When checked, Java Non-Power of Two (NPOT) Textures will be used.

Enable Concurrent Mark-Sweep Garbage Collector - When checked, Java Concurrent Mark-Sweep Garbage Collector will be used.

Java Threads

Java Rendering - The pull down menu is used to select the number of Java threads to use for rendering.

Java Reading - The pull down menu is used to select the number of Java threads to use for reading.
