Choose Cataloged Data

The General->Catalogs chooser shows THREDDS catalogs of data holdings on remote data servers (typically TDS or OPeNDAP) and provides access to remote Web Map Server (WMS) image servers. McIDAS-V provides an initial default catalog, idvcatalog.xml. It should appear in the Catalogs menu. If not you can directly enter the URL of the catalog:

http://www.unidata.ucar.edu/georesources/idvcatalog.xml

This catalog has links to a catalog of real time model data, a collection of county level shapefiles for roads and hydrography features and a collection of useful Web Map Servers.


Image 1: Catalog Chooser
To select cataloged data:
  1. In the Catalogs entry box enter the catalog URL (and press Update) or select a catalog URL from the pull down list. You can use the Select File... button to select a catalog on your local disk.
  2. Navigate through the tree view to the desired data source.
  3. Catalogs may show absolute or relative times. See below for more information.
  4. Select a data source by clicking on the particular data desired (e.g., "NCEP NAM 80km CONUS 2004-07-21 12:00:00 GMT" under "NCEP NAM 80km CONUS model data")
  5. When done, load the selected data with the Add Source button.
  6. Some catalogs contain documentation (shown under the "Documentation" nodes). You can mouse over the nodes or double click them to view the documentation.

Catalogs may list both absolute time model runs (e.g., "NCEP NAM 80km CONUS 2004-07-21 12:00:00 GMT" as well as relative time (e.g., "Latest NCEP GFS..."). You can choose to load the most recent data for each of these groups of data. When you do this McIDAS-V will query the remote server to determine what actual model run is the latest data. McIDAS-V will use that particular model throughout the entire session. However, if you save the state of the application (using the bundles facility) then when the state is loaded back in McIDAS-V will again determine what is the actual latest data and use that data set.
