7.9 Building McIDAS-V from Source
Developers can download the source files for McIDAS-V and build it from scratch.

Before building McIDAS-V you need to first install Java and Java 3D on your system. You need to use the Java SDK, because the Java RunTime Environment (JRE) does not contain the Java compiler (javac). You need to use a version of Java 1.4 or 1.5. You can use either the Java 3D SDK or JRE, version 1.3.x.

You will also need to install Ant (http://ant.apache.org/) on your machine. Ant is similar to make - instead of a Makefile, Ant uses a build.xml file. Be sure to follow the instructions for installing Ant, especially those for setting up environment variables.

Download the McIDAS-V source files from the Unidata Web Site.

  1. Open the McIDAS-V Downloads page in a new window by clicking here.
  2. To download the McIDAS-V source, you must be registered as a Unidata user. If you have already registered, log in using your user name and password. If you have not registered or you have forgotten your password, follow the instructions on the login sidebar.
  3. Under "McIDAS-V 2.4a1 Documentation and Source Code", click on the McIDAS-V 2.4a1 Source Code link to start the download. You will be prompted for a location to save the file to. Save it your home directory.
  4. You will also need the set of ancillary libraries (VisAD, netCDF, etc) that McIDAS-V uses contained in the zip file: McIDAS-V 2.4a1 JAR Files. Save this to your home directory.
  5. After the download is complete, close the extra browser window.

You are now ready to install the McIDAS-V source.

  1. Open a terminal window.
  2. Change to your home directory or the directory to which you downloaded the source and binary files (if you are not already there).
  3. unJAR the source file.

      jar  xvf  idv_src_2.4a1.jar

    This will create a "metapps" directory underneath the install directory (where you unJAR'ed the file).

  4. Next, you need to create a lib directory as a peer of the ucar directory under metapps that holds the ancillary JAR files from the McIDAS-V Binary distribution that are needed to build the classes. The structure will look like this: 
    metapps
     |
     +----ucar
     |
     +----lib (holds the ancillary jar files).
  5. Change to the lib (if you are not already there).

      cd  ~/metapps/lib

  6. Unzip the file you downloaded into this directory.

      unzip  ../../idv_jars_2.4a1.zip

Now you should be able to build McIDAS-V from scratch.
  1. Change to the ucar directory.

      cd  ~/metapps/ucar

  2. Execute the ant command to build idv.jar.

      ant  idvjar

    This removes all Java class files (there will be none the first time), re-builds them and creates the idv.jar file, copying it to the lib directory.

  3. To run McIDAS-V using your build:

    • Change to the lib directory

        cd  ../lib

    • then run:

        java  -Xmx512m  -jar idv.jar
